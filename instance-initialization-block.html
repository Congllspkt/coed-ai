Instance Initialization Block in Java

An Instance Initialization Block (IIB) in Java is a block of code that is executed when an object is created, before the constructor is called. It is used to initialize instance variables or perform any setup tasks common to all constructors.

Examples

Example 1: Using Instance Initialization Block


public class Example {
    private int value;

    // Instance Initialization Block
    {
        value = 10;
        System.out.println("Instance Initialization Block executed");
    }

    // Constructor
    public Example() {
        System.out.println("Constructor executed");
    }

    public void displayValue() {
        System.out.println("Value: " + value);
    }

    public static void main(String[] args) {
        Example example = new Example();
        example.displayValue();
    }
}

// Output:
// Instance Initialization Block executed
// Constructor executed
// Value: 10

Example 2: Multiple Instance Initialization Blocks


public class MultipleIIBs {
    private int value;

    // First Instance Initialization Block
    {
        System.out.println("First IIB executed");
    }

    // Second Instance Initialization Block
    {
        value = 20;
        System.out.println("Second IIB executed");
    }

    // Constructor
    public MultipleIIBs() {
        System.out.println("Constructor executed");
    }

    public void displayValue() {
        System.out.println("Value: " + value);
    }

    public static void main(String[] args) {
        MultipleIIBs obj = new MultipleIIBs();
        obj.displayValue();
    }
}

// Output:
// First IIB executed
// Second IIB executed
// Constructor executed
// Value: 20
