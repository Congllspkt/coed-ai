Record in Java

What is a Record?

The record keyword in Java, introduced in Java 14 as a preview feature and finalized in Java 16, is used to define a special kind of class that is immutable and primarily used to store data. Records automatically generate boilerplate code such as constructors, getters, equals, hashCode, and toString methods.

Example Usage

public record Person(String name, int age) {
}

public class RecordExample {
    public static void main(String[] args) {
        Person person = new Person("Alice", 30);
        System.out.println(person.name()); // Accessor method for name
        System.out.println(person.age()); // Accessor method for age
        System.out.println(person); // Automatically generated toString()
    }
}

In the above example, the Person record automatically generates a constructor, accessor methods, and overrides toString, equals, and hashCode.

How it Works

Records are a concise way to create data-carrying classes. They are immutable by default, meaning their fields cannot be modified after the object is created. Records are ideal for use cases where the primary purpose of a class is to store data.
