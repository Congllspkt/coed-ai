Heap Memory and Stack Memory in Java

In Java, memory is divided into two main areas: Heap Memory and Stack Memory. Understanding these areas is crucial for efficient memory management and debugging.

Heap Memory

Heap memory is used for dynamic memory allocation. Objects and their instance variables are stored in the heap. It is shared among all threads.

Key Points:

  • Used for objects and instance variables.
  • Garbage collection is performed to free unused memory.
  • Accessed globally by all threads.

Stack Memory

Stack memory is used for method execution and local variables. Each thread has its own stack memory.

Key Points:

  • Used for method calls and local variables.
  • Memory is automatically deallocated when a method ends.
  • Each thread has its own stack, making it thread-safe.

Examples

Example: Heap and Stack Memory Usage


public class MemoryExample {
    public static void main(String[] args) {
        int localVariable = 10; // Stored in stack memory
        Person person = new Person("Alice"); // Object stored in heap memory

        System.out.println("Local Variable: " + localVariable);
        System.out.println("Person Name: " + person.getName());
    }
}

class Person {
    private String name; // Instance variable stored in heap memory

    public Person(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }
}

// Output:
// Local Variable: 10
// Person Name: Alice

Example: Stack Memory for Method Calls


public class StackExample {
    public static void main(String[] args) {
        firstMethod();
    }

    public static void firstMethod() {
        secondMethod();
    }

    public static void secondMethod() {
        System.out.println("Stack memory example");
    }
}

// Output:
// Stack memory example
