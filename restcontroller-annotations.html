@RestController Annotations in Spring

What is @RestController?

The @RestController annotation in Spring is a specialized version of the @Controller annotation. It is used to create RESTful web services and combines @Controller and @ResponseBody.

Common Annotations Used with @RestController

  • @GetMapping: Handles HTTP GET requests.
  • @PostMapping: Handles HTTP POST requests.
  • @PutMapping: Handles HTTP PUT requests.
  • @DeleteMapping: Handles HTTP DELETE requests.
  • @RequestMapping: General-purpose annotation for mapping HTTP requests to handler methods.

Example of @RestController in Spring

// Example of @RestController in Spring
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestBody;
import org.springframework.web.bind.annotation.RestController;

@SpringBootApplication
public class RestControllerExample {
    public static void main(String[] args) {
        SpringApplication.run(RestControllerExample.class, args);
    }
}

@RestController
class MyController {

    @GetMapping("/greet")
    public String greet() {
        return "Hello, World!";
    }

    @PostMapping("/echo")
    public String echo(@RequestBody String message) {
        return "You said: " + message;
    }
}
